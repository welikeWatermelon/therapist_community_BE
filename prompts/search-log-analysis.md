# 작업 요청: 검색 로그 1주일 분석

## 컨텍스트

logs/search-access.log에 7일간 검색 로그 축적됨.
이 데이터를 분석하여 Level 2 (pgvector) 도입 여부를 결정하는 근거 확보.

## 분석 요청

### 1. 기본 통계
- 총 검색 수
- 고유 검색어 수
- 평균 응답 시간 (p50, p95, p99)
- 전체 zero-result rate

### 2. 검색어 패턴 분석
- 검색어 Top 20 (빈도순)
- 검색어 길이 분포 (1자, 2자, 3자, 4자, 5자 이상)
- 띄어쓰기 포함 검색어 비율

### 3. 사용자 행동 분류

검색어를 다음 카테고리로 분류:

(a) 정확 키워드형: 의학 용어, 명사형 (예: 우울증, 공황장애)
(b) 자연어 서술형: 감정/상태 서술, 문장형 (예: 힘들어요, 잠이 안 와요)
(c) 혼합형: 키워드 + 수식 (예: 요즘 우울, 30대 불안)
(d) 기타: 특수문자, 오타, 이상 입력

각 카테고리 비율 계산.

### 4. Zero-result 분석
- Zero-result 검색어 Top 20
- 각 검색어의 카테고리 (위 분류 적용)
- Zero-result 원인 추정:
    * 실제 콘텐츠 없음 (카테고리 불문)
    * 표현 차이로 놓침 (주로 카테고리 b)
    * 전처리 문제 (카테고리 d)

### 5. 판단 리포트

다음 형식으로 최종 판단:

**사용자 검색 행동 프로파일**
- (a) 정확 키워드형: XX%
- (b) 자연어 서술형: XX%
- (c) 혼합형: XX%
- (d) 기타: XX%

**현재 검색 시스템과의 정합성**
trigram이 이 프로파일에 적합한지 부적합한지 분석.

**권장 액션**

시나리오 A: trigram 유지
- 조건: (a)가 70% 이상
- 액션: Level 0 Phase 2 (word_similarity 개선)만 진행

시나리오 B: pgvector 도입
- 조건: (b)가 50% 이상 or (b)+(c)가 70% 이상
- 액션: Level 2 (pgvector) 마이그레이션 프롬프트 작성

시나리오 C: 하이브리드
- 조건: 중간 상황
- 액션: word_similarity 먼저 → 2주 재측정 → Level 2 판단
